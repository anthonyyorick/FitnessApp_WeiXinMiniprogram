# 默认数据测试说明

## 测试目的
验证应用在没有后端服务或网络连接时，能够正常显示默认数据。

## 测试步骤

### 1. 运动页面测试
1. 打开小程序，进入运动页面
2. 验证以下默认数据是否正确显示：
   - **今日数据**: 8,234步，6.2km，85分钟，320kcal
   - **运动记录**: 5条记录（跑步、步行、骑行）
3. 测试时间范围切换：
   - 点击"本周"：显示45,678步，34.5km，480分钟，1,850kcal
   - 点击"本月"：显示156,789步，118.2km，1,680分钟，6,450kcal
4. 测试运动类型筛选：
   - 点击"跑步"：只显示跑步记录
   - 点击"步行"：只显示步行记录
   - 点击"骑行"：只显示骑行记录

### 2. 健康页面测试
1. 进入健康页面
2. 验证默认数据：
   - **心率**: 65-85 bpm随机数据
   - **血压**: 110-130/70-85 mmHg随机数据
   - **睡眠**: 6-8小时随机数据
   - **体重**: 65-67kg随机数据
3. 测试健康指标切换
4. 测试时间范围切换

### 3. 首页测试
1. 进入首页
2. 验证默认数据：
   - **健康概览**: 心率75bpm，步数8,567步，睡眠良好
   - **推荐课程**: 4个示例课程
   - **健康小贴士**: 随机显示健康建议

### 4. 我的页面测试
1. 进入我的页面
2. 验证默认数据：
   - **用户信息**: 健身达人，中级，加入128天
   - **统计数据**: 总步数156,789，总里程118.2km，总消耗6,450kcal

## 测试场景

### 场景1：无网络连接
1. 关闭网络连接
2. 打开小程序
3. 验证所有页面都能正常显示默认数据

### 场景2：未登录状态
1. 确保未登录
2. 打开各个页面
3. 验证显示默认数据而不是空白页面

### 场景3：后端服务不可用
1. 确保后端服务未启动
2. 打开小程序
3. 验证网络请求失败时自动使用默认数据

## 预期结果

### ✅ 正常情况
- 所有页面都能正常显示默认数据
- 数据切换功能正常工作
- 页面布局美观，无显示错误

### ❌ 异常情况
- 页面显示空白或加载错误
- 数据切换功能失效
- 页面布局错乱

## 故障排除

### 问题1：页面显示空白
**可能原因**: 默认数据未正确加载
**解决方案**: 检查JavaScript文件中的默认数据配置

### 问题2：数据切换不生效
**可能原因**: 事件绑定或数据更新逻辑错误
**解决方案**: 检查switchTimeRange和switchSportType方法

### 问题3：样式显示异常
**可能原因**: CSS样式文件缺失或错误
**解决方案**: 检查WXSS文件是否正确配置

## 数据说明

### 运动数据格式
```javascript
{
  steps: 8234,        // 步数
  distance: 6.2,      // 距离(km)
  duration: 85,       // 时长(分钟)
  calories: 320       // 卡路里(kcal)
}
```

### 运动记录格式
```javascript
{
  id: 1,
  typeIcon: '🏃',     // 运动类型图标
  typeName: '跑步',    // 运动类型名称
  distance: '5.2',    // 距离
  duration: 32,       // 时长
  calories: 280,      // 卡路里
  startTime: '2024-01-15 07:30', // 开始时间
  type: 'running'     // 运动类型标识
}
```

### 用户信息格式
```javascript
{
  username: '健身达人',     // 用户名
  avatar: '/images/default-avatar.png', // 头像
  level: '中级',           // 等级
  joinDays: 128,          // 加入天数
  totalSteps: 156789,     // 总步数
  totalDistance: 118.2,   // 总里程
  totalCalories: 6450     // 总消耗
}
```
